= Content Studio preview
:toc: right
:imagesdir: media/

In this chapter, you'll enable Content Studio to load the preview directly from the Next.js app.

== Task: Enable preview

Install the `Next.js Preview` app that adds a new preview widget to Content Studio.

. **Install the Next.js Preview app**:
+
From `XP menu` -> `Applications` -> `Install` and search for `Next.js Preview`
+
The preview widget will work immediately after installation.
Optionally, you can add the app to the site to select a named configuration (useful for multi front-end setups, covered in the <<i18n#, Multi language>> chapter).
+
By default Next.js Preview uses the following values:
+
* *url*: `http://127.0.0.1:3000`
* *secret*: `mySecretKey`
+
This matches your setup, so everything should work out of the box. We will get back to tuning these values in the <<static#, Static pages>>, and <<deployment#, Deployment>> chapters.
+
. After applying and saving the changes, you should be able to see the live preview in Content Studio when selecting any content in the "Persons" folder.
+

== How does it work?

The Next.js Preview app adds a preview widget to Content Studio that loads pages directly from the Next.js server.
When an editor selects a content item, the widget:

. Fetches URL mapping rules from the Next.js server's `/api/mappings` endpoint
. Resolves the content item to a Next.js URL using those mappings
. Encrypts request parameters with the shared secret and sends a request
. Loads the resolved page in the preview pane

On the Next.js side, the proxy (`src/proxy.ts`) receives the request, authenticates the request by decrypting parameters, enables Next.js draft mode, and renders the page with unpublished content.

== Live page editing

In addition to preview, Content Studio supports live page editing - where editors can drag-and-drop components, edit text inline, and select components directly within the rendered page.

All you need to do as a frontend developer is to add the `PageEditorScript` component to your app, that handles all interaction!

```
<PageEditorScript mode={data.meta.renderMode}/>
```

== Link handling

Since the preview widget loads pages directly from the Next.js server, links within the rendered page need to work correctly inside Content Studio's preview pane.

The `getUrl()` function from the Enonic Adapter handles links to media files, basepath resolution, and language prefix — ensuring they work correctly both in standalone mode and in Content Studio preview.
